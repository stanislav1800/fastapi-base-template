# Auth service FastAPI

FastAPI‑проекта с авторизацией, пользователями, JWT и интеграцией с Postgres/Redis. Подходит как стартовая точка для новых сервисов.

**Что внутри**

* FastAPI + async SQLAlchemy
* JWT (access/refresh)
* Postgres + Redis
* Unit / functional / integration тесты
* Готовая Docker‑сборка

## Быстрый старт

### 1. Локальный запуск (без Docker)

1. Создайте `.env` на основе `.env.example`.
2. Установите зависимости:

```bash
uv sync
```

3. Запустите приложение:

```bash
uv run uvicorn src.main:app
```

Приложение будет доступно на `http://localhost:8000`.

### 2. Запуск через Docker

```bash
docker compose up -d --build
```

Контейнеры поднимут приложение, Postgres и Redis.

## Миграции

В Docker:

```bash
docker compose exec app uv run alembic upgrade head
```

Локально:

```bash
uv run alembic upgrade head
```

## Тесты

Локально:

```bash
pytest
```

В Docker:

```bash
docker compose exec app uv run pytest -s
```

## Импорт в новый проект

Этот шаблон рассчитан на использование как база для нового сервиса. Рекомендуемый порядок действий:

### 1. Копирование шаблона

* Склонируйте репозиторий или используйте кнопку **Use this template**.
* Переименуйте директорию проекта под ваш сервис.

### 2. Переименование проекта

* Обновите название проекта в `pyproject.toml` (`name`, `description`).
* Проверьте `project_name` / `APP_NAME` в конфигурации приложения.

### 3. Настройка окружения

* Скопируйте `.env.example` → `.env`.
* Задайте значения для:

  * подключения к Postgres (`DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`),
  * Redis,
  * JWT‑секретов и времени жизни токенов,
  * прочих параметров окружения.

### 4. Адаптация доменной логики

* Замените или удалите примерные модули (users, auth), если они не нужны.
* Пишите свои модули в `src` и подключите их в `main.py`.
* Определите собственные модели SQLAlchemy и Pydantic‑схемы.

### 5. Миграции базы данных

* Удалите примерные миграции, если схема БД будет другой.
* Создайте новые миграции через Alembic под свою модель данных.

### 6. Проверка

* Запустите приложение локально или в Docker.
* Прогоните тесты и убедитесь, что базовая инфраструктура работает корректно.

После этого шаблон можно считать полностью адаптированным под новый сервис и использовать как основу для дальнейшей разработки.
